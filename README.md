# Бизнес-функция: Покупка билетов

## Конкурентный анализ

Для разработки данной функции были изучены популярные сервисы для покупки билетов, такие как Aviasales, OneTwoTrip, Tutu.ru и Booking.com. Особое внимание уделялось тому, как пользователи ищут билеты, какие фильтры доступны, как происходит оформление заказа и получение электронного билета.

На основе проведённого анализа был сделан вывод, что пользователю важно:
- быстро находить подходящие варианты;
- удобно оплачивать заказ;
- сразу получать подтверждение покупки.

---

## Глоссарий

- **Билет** — электронный документ, подтверждающий право на поездку.
- **Пассажир** — человек, на которого оформляется билет.
- **Маршрут** — путь от пункта отправления до пункта назначения.
- **Заказ** — информация о приобретённом билете.
- **Электронный билет** — документ, который пользователь получает после успешной оплаты.
- **Бронирование** — процесс резервирования места на выбранный рейс.

---

## Бизнес-цели

- Предоставить пользователям возможность приобретать билеты через сервис туроператора.
- Упростить процесс организации поездки.
- Сократить время оформления заказа.
- Повысить количество успешных покупок.
- Улучшить пользовательский опыт.

---

## Описание бизнес-процесса

1. Пользователь выбирает пункт отправления и пункт назначения.
2. Указывает дату поездки.
3. Система отображает доступные варианты билетов.
4. Пользователь выбирает подходящий билет.
5. Вводит данные пассажира.
6. Производит оплату.
7. Система оформляет заказ.
8. Пользователь получает электронный билет и подтверждение покупки.

---

## User Story

> Как турист, я хочу купить билет через сервис туроператора, чтобы быстро и удобно организовать свою поездку в одном месте.

---

## Ограничения

- Покупка билетов доступна только авторизованным пользователям.
- Нельзя выбрать дату поездки, которая уже прошла.
- Оформление заказа возможно только при наличии свободных мест.
- Для покупки необходимо заполнить данные пассажира.
- Оплата производится банковской картой.

---

## Definition of Ready (DoR)

Задача считается готовой к разработке, если:

- определена цель функции;
- проведён анализ аналогичных сервисов;
- описан пользовательский сценарий;
- сформулированы основные требования;
- определены ограничения.

---

## Definition of Done (DoD)

Функция считается завершённой, если:

- пользователь может найти нужный билет;
- отображаются доступные варианты;
- можно выбрать билет и ввести данные пассажира;
- успешно выполняется оплата;
- после оплаты формируется заказ;
- пользователь получает электронный билет;
- функциональность протестирована и работает корректно.
